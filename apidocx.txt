Staff API (Django REST Framework)

Base URL
  /api/

Endpoints
  GET    /api/staff/                  List staff
  POST   /api/staff/                  Create staff
  GET    /api/staff/{id}/             Retrieve staff
  PUT    /api/staff/{id}/             Update staff (full)
  PATCH  /api/staff/{id}/             Update staff (partial)
  DELETE /api/staff/{id}/             Delete staff

Model Fields
  pcode          (string, read-only; generated by backend when creating)
  name           (string, required)
  address        (text, required)
  post           (string, required)
  dist           (string, required)
  state          (string, required)
  country        (string, required)
  pin            (string, required)
  joining_date   (date, optional, format: YYYY-MM-DD)
  resign_date    (date, optional, format: YYYY-MM-DD)
  status         (string, optional; choices: active, inactive; default: active)
  biodata        (file, optional)
  photo          (file, optional)
  qualification  (string, required)
  email          (string, optional)

Example: Create Staff (JSON, without files)
  POST /api/staff/
  Content-Type: application/json

  {
    "name": "Aneesh",
    "address": "Main road",
    "post": "Accountant",
    "dist": "Ernakulam",
    "state": "Kerala",
    "country": "India",
    "pin": "682001",
    "joining_date": "2025-01-01",
    "resign_date": null,
    "status": "active",
    "qualification": "B.Com",
    "email": "aneesh@example.com"
  }

Example: Create Staff (multipart, with photo & biodata)
  POST /api/staff/
  Content-Type: multipart/form-data

  name: Aneesh
  address: Main road
  post: Accountant
  dist: Ernakulam
  state: Kerala
  country: India
  pin: 682001
  joining_date: 2025-01-01
  status: active
  qualification: B.Com
  email: aneesh@example.com
  photo: <select file>
